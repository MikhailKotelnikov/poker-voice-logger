# Title

HH Manual Presupposition Fields Must Offer Click Presets

## Problem

Typing or dictating the same short presupposition markers repeatedly slows live hand review and increases inconsistency in notation.

## Rule

When rendering local or hand-level presupposition inputs in HH visualization, then show a click-select preset list (at least key defaults like `i agro`, `i gc`, `i gc+++`) while keeping manual typing and voice input available in the same control, because one-click presets speed entry without removing free-form note flexibility.

## Examples

### Positive

- Focusing a presupposition input opens a small preset menu.
- Clicking `i gc+++` appends/saves it immediately and keeps the field editable.
- User can still type custom text and use inline mic in the same field.

### Anti-pattern

- Presupposition inputs require full manual typing for repeated standard markers.
- Preset selection replaces and blocks free-form typing or disables voice continuation.

## Validation Checklist

- [ ] Specific and testable
- [ ] Reusable in future tasks
- [ ] Not duplicated elsewhere
